# Codex Rules

## OpenAI Documentation

When answering questions about OpenAI APIs, models, Codex, SDKs, or OpenAI developer documentation, use the `openaiDeveloperDocs` MCP server first. Prefer official OpenAI docs over memory or general web search, and fetch the relevant documentation page before giving exact API, schema, parameter, model, or migration guidance.

## Workspace Guidelines

- Read existing project files before making changes.
- Keep edits focused on the requested task.
- Do not overwrite user changes unless explicitly asked.
- Prefer fast search tools such as `rg` when inspecting the repository.
- Verify changes when practical before reporting completion.
