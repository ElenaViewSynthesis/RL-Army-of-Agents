# Project Rules

## Documentation Sources

Use official OpenAI documentation as the source of truth for OpenAI APIs, models, SDKs, Codex, and migration guidance.

## Development Practices

- Keep changes small, clear, and scoped to the task.
- Follow existing project conventions once the project has code.
- Avoid unrelated refactors.
- Preserve user work and do not revert changes without permission.
- Run relevant checks or tests when available.
