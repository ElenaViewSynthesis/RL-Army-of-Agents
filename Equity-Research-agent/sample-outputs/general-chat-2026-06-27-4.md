### 2. Java NIO (New I/O / Non-blocking I/O)
*   **What it is:** A collection of Java programming APIs (introduced in Java 1.4, updated in Java 7 with **NIO.2**) for intensive I/O operations.
*   **Core Components:**
    *   **Buffers:** Containers for data (e.g., `ByteBuffer`, `CharBuffer`).
    *   **Channels:** Connections to I/O devices (e.g., `FileChannel`, `SocketChannel`, `ServerSocketChannel`).
    *   **Selectors:** Multiplexing mechanism allowing a single thread to manage multiple channels (non-blocking I/O).
*   **Key Benefit:** Enables high-scalability network servers (handling thousands of connections with few threads) compared to the traditional blocking `java.io` (BIO).

### 3. Linux `io_uring` / Kernel Async I/O
*   Sometimes referred to in systems programming contexts as "Linux NIO" or "Async I/O," though the standard term is usually **`io_uring`** (modern) or **AIO** (legacy `libaio`). It allows applications to perform I/O without blocking syscalls.
